1Ô∏è‚É£6Ô∏è‚É£ Difference between Offset and Cursor pagination?

Definition:
Offset pagination retrieves data by skipping a specific number of records (OFFSET) and then limiting the result (LIMIT).

üëâ It is page-number‚Äìbased.

Example:

SELECT * FROM users
ORDER BY id
LIMIT 10 OFFSET 20;

‚û°Ô∏è Skips first 20 records and returns next 10.

üîπ Cursor Pagination

Definition:
Cursor pagination retrieves data based on a reference point (cursor), such as the last seen id or timestamp.

üëâ It is pointer-based, not page-number‚Äìbased.
Example:
SELECT * FROM users
WHERE id > 20
ORDER BY id
LIMIT 10;
‚û°Ô∏è Fetches next 10 records after id = 20.

| Feature          | Offset Pagination                   | Cursor Pagination       |
| ---------------- | ----------------------------------- | ----------------------- |
| Pagination Type  | Page-based                          | Cursor-based            |
| Performance      | Slower for large datasets           | Very fast               |
| Data Consistency | ‚ùå Can cause duplicates/missing data | ‚úÖ Consistent            |
| Use Case         | Small datasets, admin panels        | Large-scale apps, feeds |
| Implementation   | Simple                              | Slightly complex        |

üîπ ADVANCED REST API QUESTIONS -----
2Ô∏è‚É£0Ô∏è‚É£ REST vs SOAP?
üîπ REST (Representational State Transfer)
Definition:
REST is an architectural style for building web APIs that uses standard HTTP methods to perform operations on resources.

Key Idea:
üëâ Everything is a resource (users, orders, products) identified by URLs.

Example (REST API):

GET /api/users/10 HTTP/1.1
Host: example.com


Response (JSON):

{
  "id": 10,
  "name": "Palash",
  "email": "palash@example.com"
}

üîπ SOAP (Simple Object Access Protocol)

Definition:
SOAP is a strict protocol for exchanging structured information using XML with predefined standards.

Key Idea:
üëâ Follows WSDL contract and strict rules.

Example (SOAP Request):

<soap:Envelope>
  <soap:Body>
    <GetUser>
      <UserId>10</UserId>
    </GetUser>
  </soap:Body>
</soap:Envelope>


Response (XML):

<GetUserResponse>
  <User>
    <Id>10</Id>
    <Name>Palash</Name>
  </User>
</GetUserResponse>


| Feature     | REST                | SOAP               |
| ----------- | ------------------- | ------------------ |
| Type        | Architectural style | Protocol           |
| Data Format | JSON (mostly)       | XML only           |
| Performance | Fast, lightweight   | Slower, heavy      |
| Ease of Use | Easy                | Complex            |
| Security    | HTTPS, OAuth, JWT   | WS-Security        |
| Flexibility | High                | Low (strict rules) |
| Contract    | Optional            | Mandatory (WSDL)   |
